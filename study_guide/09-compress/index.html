1 Description

It’s time to reduce bandwidth comsumption in InterCom. The pack() and the unpack() methods can compress and decompress, respectively, the chunks that are processed. To compress and decompress, we will use a free codec named DEFLATE, which is based on LZSS and Huﬀman Coding [1]. See this notebook and this notebook.

The DEFLATE algorithm is implemented in the Python’s standard library zlib. We have used this facility for compressing and decompressing the chunks that we are sending and receiving in the methods pack() and unpack(), respectively. These methods are implemented in the modules compress{|0|1|2}.py, inside of a the class Compression{|0|1|2} (respectively), and override the deﬁned in the parent class:

  1. compress.py: Implements no compression at all.
  2. compress0.py: Compress the raw chunks with DEFLATE.
  3. compress1.py: Compress the chunk after concatenating the channels (see Fig. 1). Notice that with this shuﬄing, the samples are not interleaved and the correlation between consecutive bytes is slighly increased. This should increase the (data) compression ratio


    Figure 1: Sample reordering to create two independent channels.
  4. compress2.py: similar to compress1.py, but reseting DEFLATE at each new chunk-channel.

Finally, notice that the number of sent UDP packets remain constant respect to parent code.
