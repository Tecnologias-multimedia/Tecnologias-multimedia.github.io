1.1 Overlapped block transforms for minimizing the distortion

Transform coding implies to split the signal into blocks of data (chunks), and to compute the transform of each chunk. When the output coeﬃcients are quantized, it is possible that signiﬁcative (and unpleasant) distortions may appear between the border frames of the chunks (see the Fig 1). This is a consequence of that in the prediction step used by the DWT in the limits of the chunks generate diﬀerent predictions at both sides of the border.


Figure 1: On the top-left, three consecutive chunks of a real mono audio sequence. On the top-right, the reconstruction of the chunks without overlapping. On the bottom-left, the extended central chunk. On the bottom-right, the reconstruction of the extended chunk. See this notebook.

One solution to avoid signal discontinuitites between chunks is to overlap the chunks. Thus, the current (i-th) chunk uses also the last frames of the previous ((i − 1)-th) chunk and the ﬁrst frames of the next ((i + 1)-th) chunk to compute the transform of the current extended (i-th) chunk (see the Fig. 2). This has been described in the following algorithm:

Encoder:

  1. C−1 = 0.
  2. Input C1.
  3. For i ∈{1,2,⋯}:
    1. Input chunk Ci+1.
    2. Compute decomposition Di ←DWTl(Ci−1[−o :]|Ci|Ci+1[: o]), where Di is the decomposition of the i-th extended chunk, ⋅|⋅ denotes the concatenation-of-frames operator, deﬁned by
      a|b = {a,b}, (1)

      and Ci−1[−o :]|Ci|Ci+1[: o] is the i-th extended chunk (see Fig. 2), where o is the overlapped area size in frames, C[: o] are the ﬁrst o frames of the chunk C, C[−o :] its last o frames, and l is the number of levels of the DWT (l = 2 in the Fig. 2).

    3. Send decomposition Di.
    4. Ci−1 ← Ci.
    5. Ci ← Ci+1.

Notice that we are sending for each chunk 2o + x stereo-coeﬃcients, where x is the number of frames/chunk.

Decoder:

  1. For i ∈{0,1,⋯}:
    1. Receive decomposition Di.
    2. Compute extended chunk Ci ←DWT−l(Di).
    3. Output chunk Ci[o : −o].


Figure 2: Structure in the DWT domain of an extended chunk for l = 2. o is the number of frames to overlap between adajacent chunks. C[: o] represents the ﬁrst o framees of the chunk C and C[−o :] the last x − o frames of chunk C, where x is the number of frames in C.

This idea has been implemented in this notebook, and the result can be seen in the Fig. 1.

1.2 Reducing the data overhead

Unfortunately, the previous algorithm sends twice the DWT coeﬃcients of the overlapped areas (in the Fig. 2, {Ci−1[−o :].L2,Ci+1[: o].L2,Ci−1[−o :].H2,Ci+1[: o].H2,Ci−1[−o :].H1,Ci+1[: o].H1}). To avoid this waste of bandwidth, we can reuse the received coeﬃcients of the overlapped areas. This procedure has been described in the Fig. 3. As it can be seen, the encoding algorithm is identical to the previous one except in that only the central (stereo) coeﬃcients are sent. The rest of coeﬃcients that are needed to compute the inverse transform are extracted from the neighbor chunks (represented in the DWT domain). Notice that now, the number of sent coeﬃcients is x.


Figure 3: Block overlapping in the DWT domain for l = 2. Only the shady coeﬃcients are transmitted.

The codec now can be described by:

Encoder:

  1. For i ∈{0,1,⋯}:
    1. Input chunk Ci.
    2. Compute decomposition Di ←DWTl(Ci).
    3. Send Di.

Decoder:

  1. D−1 = 0.
  2. Input decomposition D1.
  3. For i ∈{1,2,⋯}:
    1. Receive decomposition Di+1.
    2. Build extended decomposition (l = 2) Ei = Di−1[−o :].L2|Di.L2|Di+1[: 0].L2|Di−1[−o :].H2|Di.H2|Di+1[: 0].H2|Di−1[−o :].H1|Di.H1|Di+1[: 0].H1.
    3. Compute extended chunk Ci ←DWT−l(Ei).
    4. Output Ci[o : −o].
    5. Di−1 ← Di.
    6. Di ← Di+1.
