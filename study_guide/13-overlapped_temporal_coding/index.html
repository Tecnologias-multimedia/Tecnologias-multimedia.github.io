1 Description

1.1 Overlapped block transforms for minimizing the distortion

Transform coding implies to split the signal into blocks of data (chunks), and to compute the transform of each chunk. When the output coeﬃcients are quantized, it is possible that signiﬁcative (and unpleasant) distortions may appear in the border frames of the chunks (see Fig 1). This is a consequence of that in the prediction step used by the DWT in the limits of the chunks generate diﬀerent predictions at the beginning and the end of the chunks.


Figure 1: On the top-left, three consecutive chunks of a real mono audio sequence. On the top-right, the reconstruction of the chunks without overlapping. On the bottom-left, the extended central chunk. On the bottom-right, the reconstruction of the extended chunk. See this notebook.

One solution to avoid signal discontinuitites between chunks is to overlap the chunks. Thus, the current (i-th) chunk uses also the last frames of the previous ((i − 1)-th) chunk and the ﬁrst frames of the next ((i + 1)-th) chunk to compute the transform of the current extended (i-th) chunk (see the Fig. 2). This has been described in the following algorithm:

Encoder:

  1. C−1 = 0, a zero chunk.
  2. Input C0.
  3. For i ∈{0,1,⋯}:
    1. Input Ci+1.
    2. Build extended chunk E ←Ci−1[−o :]|Ci|Ci+1[: o], where ⋅|⋅ denotes the concatenation of chunks, o is the overlapped area size in frames, Ci−1[−o :] the last o frames of chunk Ci−1, and Ci+1[: o] are the ﬁrst o frames of the chunk Ci+1.
    3. Compute decomposition Di ←DWTl(E), where l is the number of levels of the DWT (l = 2 in the Fig. 2).
    4. Output decomposition Di.
    5. Ci−1 ←Ci.
    6. Ci ←Ci+1.

Decoder:

  1. For i ∈{0,1,⋯}:
    1. Input decomposition Di.
    2. Compute extended chunk E ←DWT−l(Di).
    3. Output chunk Ci = E[o : −o].


Figure 2: Structure in the DWT domain of an extended chunk for l = 2. o is the number of overlapped frames between adajacent chunks. Ci−1[−o :] represents the last o frames of chunk Ci−1, and Ci+1[: o] the ﬁrst o frames of the chunk Ci+1.

This idea has been implemented in this notebook, and the result can be seen in the Fig. 1.

1.2 Reducing the data overhead

Unfortunately, the previous algorithm sends twice the DWT coeﬃcients of the overlapped areas (in the Fig. 2, {Di.l2[−o∕4 :],Di.l2[: o∕4],Di.h2[−o∕4 :],Di.h2[: o∕4],Di.h1[−o∕2 :],Di.h1[: o∕2]}). To avoid this waste of bandwidth, we can reuse the received coeﬃcients of the overlapped areas. This procedure has been described in the Fig. 3, and, it can be seen, the encoding algorithm is identical to the previous one except in that only the central (stereo) coeﬃcients are sent. The rest of coeﬃcients that are needed to compute the inverse transform are extracted from the neighbor chunks (represented in the DWT domain). Notice that now, the number of sent coeﬃcients is len(Ci), the number of samples in Ci.


Figure 3: Block overlapping in the DWT domain for l = 2. Only the shadded coeﬃcients are transmitted. Notice that, to be reconstructed, each chunk depends on some coeﬃcients of the adjacent blocks (only some dependencies have been indicated).

The codec now can be described by:

Encoder:

  1. C−1 = 0, a zero chunk.
  2. Input C0.
  3. For i ∈{0,1,⋯}:
    1. Input Ci+1.
    2. Build extended chunk E ←Ci−1[−o :]|Ci|Ci+1[: o].
    3. Compute decomposition Di ←DWTl(E).
    4. Output {Di.ll[ o 2l],Di.hl[ o 2l],Di.hl−1[ o 2l−1 ],⋯,Di.h1[o 2] }.
    5. Ci−1 ←Ci.
    6. Ci ←Ci+1.

Decoder:

  1. For i ∈{0,1,⋯}:
    1. Input decomposition Di.
    2. Compute chunk Ci ←DWT−l(Di).
    3. Output Ci.
